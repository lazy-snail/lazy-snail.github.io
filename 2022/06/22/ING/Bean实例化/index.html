实例化方法

构造器方式

即采用无参、有参构造方法实例化对象。一般使用@Resource、@Component等注解即可。

静态工厂方式

创建一个学生类:

学生类 
package io.neil.ioc;

public class Student {
	private String name;
	private int age;

	public Student(String name) {
		this.name = name;
	}

	public String getName() {
		return name;
	}

	public void setName(String name) {
		this.name = name;
	}

	public int getAge() {
		return age;
	}

	public void setAge(int age) {
		this.age = age;
	}
}

创建一个学生工厂类，在静态方法上添加@Bean：

学生工厂类 
package io.neil.ioc;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;

public class StudentFactory {
	@Bean(name = "student")
	public static Student getStudent(@Value("Neil") String name) {
		return new Student(name);
	}
}

创建一个配置类，使用@Import注解导入静态工厂方法类：

配置类 
package io.neil.ioc;

import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Import;

@Configuration
@Import(StudentFactory.class)
public class BeanConfig {

}

实例工厂方式

同学类同上，工厂类的不同在于这次不是静态方法：

学生工厂类 
package io.neil.ioc;

public class StudentFactory2 {
	public Student getStudent(String name) {
		return new Student(name);
	}
}

配置信息：

配置信息 
<!--	定义实例工厂bean-->
<bean id="studentInstanceFactory" class="io.neil.ioc.StudentFactory2"/>
<!--	使用实例工厂bean创建bean-->
<bean id="student2" factory-bean="studentInstanceFactory" factory-method="getStudent">
    <constructor-arg index="0" value="Neil N"></constructor-arg>
</bean>

反射：最常用的实例化方法

反射机制是java本身比较底层的一种技术，可以参考这里：java反射
一般情况下，Spring通过反射机制实现类对象的实例化。以注解配置为例，spring就是通过读取特定的注解，如@Component、@Service等，把类的元信息：class、属性名、属性值等，通过反射创建出对象，然后把它们放在map中，key就是对应的id或者name，即所谓的IOC容器，也就是上面提到的BeanFactory，当某个地方需要用到一个bean时，就可以直接通过key来获取，简化意义上的IOC就是这样。
